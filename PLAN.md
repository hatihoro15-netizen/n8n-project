# n8n 자동화 기획서 — AI 멀티 에이전트 영상 제작 시스템

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 목적 | n8n 안에서 AI 에이전트들이 각자 역할을 맡아 유튜브 영상을 **기획→제작→업로드→분석**까지 자동 수행 |
| 핵심 개념 | 에이전트 = 부서. 사람이 아닌 **AI가 각 부서 역할**을 하며, 서로 의견을 주고받고 자율적으로 동작 |
| 최종 목표 | **내가 안 해도 돌아가는 시스템** — 모니터링만 하면 됨 |

### 운영 중인 사업

| # | 사업/채널 | 콘텐츠 유형 | 비고 |
|---|----------|------------|------|
| 1 | 온카스터디 커뮤니티 | 롱폼 + 숏폼 | 커뮤니티 운영 내용을 영상화 |
| 2 | 슬롯 채널 | 롱폼 | |

| 4 | 쇼츠 전용 채널 | 숏폼 (쇼츠) | 다양한 소재 + 위 사업들 광고 삽입 |

## 2. 에이전트(부서) 구성

```
┌─────────────────────────────────────────────────────┐
│                    총괄 매니저 (PM)                     │
│          전체 흐름 관리, 에이전트 간 조율, 최종 승인          │
└──────────────────────┬──────────────────────────────┘
                       │
       ┌───────────────┼───────────────┐
       ▼               ▼               ▼
  ┌─────────┐   ┌──────────┐   ┌──────────┐
  │ 데이터    │   │ 기획팀    │   │ 제작팀    │
  │ 분석팀   │──▶│          │──▶│          │
  │ (DA)    │   │ (PL)     │   │ (PR)     │
  └─────────┘   └──────────┘   └──────────┘
       ▲               ▲               │
       │               │               ▼
       │          ┌──────────┐   ┌──────────┐
       │          │ 검수/QA팀  │◀──│ 배포팀    │
       │          │ (QA)     │   │ (DL)     │
       │          └──────────┘   └──────────┘
       │               │               │
       └───────────────┴───────────────┘
              피드백 루프 (순환)
```

### 에이전트 상세 역할

| 에이전트 | 약칭 | 역할 | AI 모델 | 핵심 행동 |
|---------|------|------|---------|----------|
| **총괄 매니저** | PM | 전체 파이프라인 관리, 에이전트 간 조율 | GPT-4 / Claude | 작업 지시, 우선순위 결정, 최종 승인/반려 |
| **데이터 분석팀** | DA | 유튜브 성과 분석, 트렌드 파악 | GPT-4 | 조회수/CTR/유지율 분석 → "이런 영상 만들어야 함" 리포트 생성 |
| **기획팀** | PL | 영상 아이디어 생성, 대본/구성안 작성 | Claude / GPT-4 | DA 리포트 기반으로 기획안 2~3개 생성, 서로 다른 방향 제시 |
| **검수/QA팀** | QA | 기획안 검토, 피드백, 품질 관리 | Claude | 기획안 평가 → 통과/수정요청/반려 판단, 구체적 피드백 제공 |
| **제작팀** | PR | 영상 실제 제작 (스크립트→영상) | AI 영상 도구 | 대본 기반 TTS + 이미지/영상 소스 조합 → 쇼츠/롱폼 생성 |
| **배포팀** | DL | 유튜브 업로드, 썸네일, 태그, 설명 | GPT-4 | 제목/설명/태그 최적화 → YouTube API로 업로드 → 광고 삽입 관리 |

## 3. 에이전트 간 워크플로우 (전체 흐름)

### 3-1. 메인 파이프라인: 영상 자동 생산

```
[매일 자동 실행]

① DA (데이터 분석)
   │  YouTube Analytics API로 전체 채널 성과 수집
   │  → 조회수 높은 영상 패턴 분석
   │  → 트렌드 키워드 수집
   │  → "분석 리포트" 생성
   ▼
② PL (기획)
   │  분석 리포트를 받아서 영상 기획안 생성
   │  → 기획안 A (안전한 방향)
   │  → 기획안 B (실험적 방향)
   │  → 각 기획안에 대본 초안 포함
   ▼
③ QA (검수)
   │  기획안 A, B를 평가
   │  → 점수 매기기 (참신함, 조회수 예측, 브랜드 적합도)
   │  → 통과 → 제작팀으로 전달
   │  → 수정 필요 → 구체적 피드백과 함께 기획팀에 반환
   │     └→ PL이 수정 → QA 재검토 (최대 3회 반복)
   ▼
④ PR (제작)
   │  승인된 기획안으로 영상 제작
   │  → TTS 음성 생성 (ElevenLabs / OpenAI TTS)
   │  → AI 이미지/영상 생성 (Runway / Pika / Creatomate)
   │  → 편집 + 자막 + BGM 합성
   │  → 광고 삽입 (온카스터디/루믹스 등 자동 배치)
   ▼
⑤ DL (배포)
   │  → 제목 / 설명 / 태그 AI 최적화
   │  → 썸네일 자동 생성
   │  → YouTube API로 업로드 (예약 발행)
   │  → 업로드 완료 알림
   ▼
⑥ DA (다시 분석)
   │  업로드 후 24시간/48시간/7일 성과 추적
   │  → 결과를 다음 기획 사이클에 반영
   └→ ①로 돌아감 (무한 루프)
```

### 3-2. 피드백 루프 (에이전트 간 토론)

```
기획팀(PL) ←→ 검수팀(QA) 토론 프로세스:

PL: "이 기획안 어때?"
    ▼
QA: 평가 (1~10점)
    ├─ 7점 이상 → ✅ 승인, 제작팀으로
    ├─ 4~6점 → 🔄 "여기 수정해" + 구체적 피드백 → PL에게 반환
    │           └→ PL 수정 후 재제출 (최대 3라운드)
    └─ 3점 이하 → ❌ 폐기, 새 기획안 요청

이 과정은 n8n의 Loop/If 노드로 구현
```

## 4. 채널별 콘텐츠 전략

### 4-1. 콘텐츠 유형 매트릭스

| 채널 | 숏폼 (쇼츠) | 롱폼 | 광고 삽입 | 발행 빈도 |
|------|-----------|------|----------|----------|
| 온카스터디 | O | O | 자체 홍보 | 주 3~5회 |
| 슬롯 | - | O | 온카/루믹스 광고 | 주 2~3회 |
| 루믹스 솔루션 | O | O | 자체 홍보 | 주 3~5회 |
| 쇼츠 전용 | O | - | 온카/루믹스/기타 광고 | 매일 1~3회 |

### 4-2. 광고 삽입 규칙

| 규칙 | 내용 |
|------|------|
| 쇼츠 광고 비율 | 3개 중 1개는 온카스터디/루믹스 광고 포함 |
| 롱폼 광고 | 영상 시작 or 중간에 5~10초 광고 삽입 |
| 광고 소재 | 미리 등록된 광고 소재 풀에서 자동 선택 |
| 로테이션 | 같은 광고가 연속 나오지 않도록 자동 관리 |

## 5. 필요 도구 및 API

### 5-1. AI 모델

| 용도 | 도구 | 비고 |
|------|------|------|
| 텍스트 생성 (기획/대본/분석) | OpenAI GPT-4 / Claude API | n8n AI 노드 활용 |
| TTS (음성 합성) | ElevenLabs / OpenAI TTS | 나레이션 자동 생성 |
| 이미지 생성 | Midjourney API / DALL-E / Flux | 썸네일 + 영상 내 이미지 |
| 영상 생성/편집 | Creatomate / Shotstack / Runway | API로 영상 자동 조합 |

### 5-2. 외부 서비스

| 서비스 | 용도 | API |
|--------|------|-----|
| YouTube Data API | 성과 분석 + 업로드 | OAuth2 |
| YouTube Analytics API | 조회수/CTR/유지율 데이터 | OAuth2 |
| Google Sheets | 데이터 저장/관리 | OAuth2 |
| Google Trends | 트렌드 키워드 수집 | 비공식 API / SerpAPI |
| Slack / Discord / 카카오톡 | 알림 수신 | Webhook / Bot Token |

### 5-3. 저장소

| 용도 | 도구 |
|------|------|
| 영상 파일 임시 저장 | Google Drive / S3 |
| 기획안/대본 저장 | Google Sheets (DB 역할) |
| 광고 소재 풀 | Google Drive 폴더 |
| 실행 로그 | n8n 내장 + Google Sheets |

## 7. 구축 로드맵

### Phase 1 — 기반 세팅 (1주)

- [ ] Hostinger VPS 구매 + n8n 설치
- [ ] YouTube API, OpenAI API, TTS API Credential 등록
- [ ] Google Sheets DB 구조 설계 (영상 DB, 기획안 DB, 성과 DB)
- [ ] 광고 소재 풀 Google Drive에 정리

### Phase 2 — 데이터 분석 에이전트 (1~2주)

- [ ] DA-01: YouTube Analytics 데이터 자동 수집
- [ ] DA-02: 트렌드/키워드 분석
- [ ] DA-03: AI 분석 리포트 자동 생성
- [ ] 검증: 리포트 품질 확인 및 프롬프트 튜닝

### Phase 3 — 기획 + 검수 에이전트 (2~3주)

- [ ] PL-01: AI 기획안 자동 생성
- [ ] QA-01: 기획안 자동 검수 + 피드백
- [ ] PL-02: 피드백 반영 수정 루프
- [ ] 검증: 기획안 품질이 실제 사용 가능한 수준인지 확인

### Phase 4 — 제작 에이전트 (3~4주)

- [ ] PR-01: 스크립트 확정
- [ ] PR-02: TTS 음성 자동 생성
- [ ] PR-03: 영상 자동 제작 (쇼츠부터 시작)
- [ ] PR-04: 광고 자동 삽입
- [ ] 검증: 영상 품질 확인 및 파이프라인 안정화

### Phase 5 — 배포 + 총괄 연결 (1~2주)

- [ ] DL-01~03: 메타데이터 생성 + 업로드 + 알림
- [ ] PM-01: 전체 파이프라인 연결
- [ ] PM-02: 에러 처리 / 재시도 로직
- [ ] 전체 E2E 테스트

### Phase 6 — 고도화 (지속)

- [ ] 롱폼 영상 제작 파이프라인 추가
- [ ] 채널별 스타일/톤 분리
- [ ] 성과 기반 자동 학습 (잘된 영상 패턴 → 기획에 반영)
- [ ] A/B 테스트 자동화 (같은 주제, 다른 썸네일/제목)

## 8. 주의사항 및 제약

| 항목 | 내용 |
|------|------|
| YouTube API 할당량 | 일일 10,000 유닛 제한 — 업로드는 1건당 1,600 유닛 소모 (일 최대 6건) |
| AI 비용 | GPT-4 / Claude API 호출 비용 발생 — 기획+검수 1사이클당 약 $0.1~0.5 |
| TTS 비용 | ElevenLabs 월 $5~22 (캐릭터 수 기반) |
| 영상 제작 도구 | Creatomate 월 $39~ / Shotstack 종량제 — 비용 모니터링 필요 |
| 영상 품질 | 초기에는 AI 제작 영상 품질이 낮을 수 있음 — 쇼츠부터 시작 권장 |
| 저작권 | AI 생성 이미지/음악 저작권 확인 필요 |
