## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 아럽토 유튜브 숏츠 자동화 |

## 3. 콘텐츠 카테고리 (7개)

### 카테고리 1: 프로토 배당 분석 ⭐ 우선순위 1위

**컨셉**: 프로토 배당률 활용 수익 시뮬레이션, 고배당 경기 분석

**영상 구조**:
- 도입(후킹): "1000원으로 32000원 벌 수 있다고?" / "오늘 고배당 경기 TOP3"
- 본문(정보): 배당률 설명 → 환급금 계산 과정 → 조합 전략
- CTA: "이 배당 지금 라이브365에서 실시간 확인 가능!"

**슬라이드 구성 (5~6장)**:
1. 후킹 텍스트 + 배당률 표 캡처
2. 경기 매치업 이미지
3. 배당률 계산 과정 (환급금 = 배당률 × 투표금액)
4. 조합 전략 비교
5. 수익 시뮬레이션 결과
6. CTA — 라이브365 /proto 페이지 캡처

**장점**: 수익 관련 → 후킹력 최강

---

### 카테고리 2: 오늘의 경기 분석/예측 ⭐ 우선순위 2위

**컨셉**: 주요 경기 양팀 비교, 승/무/패 예측

**영상 구조**:
- 도입: "맨시티 vs 리버풀, 오늘 누가 이길까?"
- 본문: 양팀 전적 → 최근 폼 → 상대전적 → 핵심선수 비교
- CTA: "경기 분석 더 보려면? 라이브365 프리미엄분석!"

**슬라이드 구성 (5~6장)**:
1. VS 매치업 이미지
2. 양팀 최근 5경기 전적
3. 상대 전적 비교
4. 핵심 선수/부상자 정보
5. 최종 예측
6. CTA — 라이브365 /analysis 캡처

**장점**: 매일 새 콘텐츠 생산 가능, 경기 전 검색 수요 높음

---

### 카테고리 3: 경기 결과 속보 ⭐ 우선순위 3위

**컨셉**: 당일 주요 경기 결과 빠른 요약

**영상 구조**:
- 도입: "오늘 EPL 결과 한눈에!"
- 본문: 주요 스코어 + 이변/하이라이트 포인트
- CTA: "모든 경기 결과 실시간으로? 라이브365 하나면 끝!"

**슬라이드 구성 (4~5장)**:
1. 후킹 텍스트 + 리그 로고

5. CTA — 라이브365 홈 화면

**장점**: 속보성 → 알고리즘 타기 좋음

## 5. 영상 제작 파이프라인

### 제작 흐름
```
Google Sheets(주제) → Gemini AI(스크립트 생성) → 파싱
  → [병렬] TTS 생성 + 이미지 준비(사이트 캡처 + Pexels)
  → 데이터 병합 → NCA FFmpeg 합성 → 시트 상태 업데이트
```

### 기존 onca_shortform과 차이점

| 항목 | onca_shortform (기존) | 아럽토 (신규) |
|------|----------------------|---------------|
| 이미지 | Pexels 스톡 | 라이브365 캡처 |
| CTA | 온카스터디 | 라이브365 |
| 주제 | 카지노/먹튀검증 | 스포츠 정보/배당 |
| 프롬프트 | 기존 프롬프트 | 신규 설계 필요 |
| BGM | 교육/경고 | 스포츠/에너지틱 |

### 이미지 전략
- **1차**: 라이브365 사이트 캡처 (Puppeteer/Playwright 자동화)
- **2차**: Pexels 보조 이미지 (경기장, 스포츠 분위기 연출)

## 6. 제작 우선순위

| 순위 | 카테고리 | 이유 | 난이도 |
|------|----------|------|--------|
| 1 | 프로토 배당 분석 | 후킹력 최강 | 보통 |
| 2 | 경기 분석/예측 | 매일 생산, 검색 수요 | 보통 |
| 3 | 경기 결과 속보 | 속보성, 빠른 제작 | 쉬움 |
| 4 | 적중인증/커뮤니티 | 바이럴, 신뢰도 | 쉬움 |
| 5 | 초보자 가이드 | 신규 유입 | 쉬움 |
| 6 | 멀티종목 비교 | 차별화 | 보통 |
| 7 | 사이트 기능 소개 | 직접 홍보 | 쉬움 |

### 개발 단계
- **Phase 1** — 기반 구축: 프롬프트 설계 + 시트 생성 + 캡처 자동화 + n8n 워크플로우
- **Phase 2** — 1순위 자동화: 프로토 배당 영상 완성 + 테스트 3개 + BGM 선정
- **Phase 3** — 확장: 경기 분석 + 결과 속보 추가 + 카테고리 로테이션
- **Phase 4** — 완성: 전체 카테고리 + API/크롤링 연동 + 업로드 자동화

## 8. 참고사항

- **저작권**: 사이트 소유자 = 프로젝트 운영자 → 전체 자유 활용
- **팀 로고 주의**: 구단 저작권이므로 화면 전체 캡처 방식 사용
- **기존 연계**: onca_shortform 워크플로우 구조 최대한 재활용
- **향후 확장**: 슬라이드 안정화 후 VS대결형, 시뮬레이션형, 데모형 순차 도입
