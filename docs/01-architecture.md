## 데이터 흐름
```
[스케줄 트리거] → [AI 콘텐츠 생성 (Gemini)] → [콘텐츠 파싱]
    → [시트 기록 (Google Sheets)] → [참조 이미지 프롬프트]
    → [세그먼트 분리] → [이미지 생성 (Kie.ai)]
    → [영상 순차 생성 (Kie.ai Veo3)] × 3장면
    → [NCA 영상 합성 (FFmpeg)] → [상태 업데이트]
```

## 에이전트 팀 시스템

### 역할 정의
| 에이전트 | 담당 영역 | 수정 가능 파일 |
|---------|-----------|---------------|
| 인프라 에이전트 | 서버, 배포, n8n 인스턴스 관리 | 서버 설정 관련 파일만 |
| 워크플로우 에이전트 | 노드 구성, 노드 간 연결, 워크플로우 로직 | *.json 워크플로우, *.py 생성 스크립트 |
| 콘텐츠 에이전트 | AI 프롬프트, 캐릭터 설정, 대사 작성 | 프롬프트 텍스트, 캐릭터 설정값, 콘텐츠 관련 노드 |
| 검증 에이전트 | 테스트, lint, 동작 확인 | 없음 (읽기 전용) |

### 에이전트 규칙
- 자기 역할 외 파일 수정 금지
- 작업 전 반드시 관련 docs 파일 읽기
- 작업 완료 후 반드시 보고서 작성:
  - 뭘 발견했는지
  - 뭘 수정했는지
  - 왜 그렇게 판단했는지
  - 수정된 파일 목록
  - 검증 결과
