# 01. 시스템 아키텍처

## 시스템 구조
- **n8n 워크플로우 엔진**: 자동화 파이프라인 관리
- **Python 스크립트**: 워크플로우 생성/수정 도구
- **JSON 워크플로우 파일**: n8n 워크플로우 정의

## 서버 구성
- **n8n 서버**: https://n8n.srv1345711.hstgr.cloud
- **로컬 개발**: ~/n8n-project

## 데이터 흐름
```
[Python 스크립트] → [JSON 워크플로우 생성]
        ↓
[n8n 서버 업로드] → [워크플로우 실행]
        ↓
[YouTube API] → [광고 콘텐츠 자동 생성/관리]
```

## 디렉토리 구조
```
n8n-project/
├── CLAUDE.md           # 작업 규칙 (최우선)
├── PROGRESS.md         # 작업 기록
├── docs/               # 매뉴얼 문서
├── scripts/            # 유틸리티 스크립트
├── .claude/commands/   # Claude 커맨드
├── *.py                # 워크플로우 생성 스크립트
├── *.json              # n8n 워크플로우 파일
└── *.xlsx              # 기획서/보고서
```

## 에이전트 역할 정의

### 인프라 에이전트
- 담당: 서버, DB, 배포 관련 파일
- 권한: 서버 설정, n8n 인스턴스 관리

### 워크플로우 에이전트
- 담당: n8n 노드 구성, 노드 간 연결, 워크플로우 로직
- 권한: *.json 워크플로우 파일, *.py 생성 스크립트

### 콘텐츠 에이전트
- 담당: AI 프롬프트, 캐릭터 설정, 대사 작성
- 권한: 프롬프트 텍스트, 캐릭터 설정값, 콘텐츠 관련 노드

### 검증 에이전트
- 담당: 테스트, lint, 동작 확인
- 권한: scripts/, 테스트 파일

### 에이전트 공통 규칙
- 자기 역할 외 파일 수정 금지
- 작업 전 반드시 관련 docs 파일 읽기
- 작업 완료 후 반드시 보고서 작성:
  - 뭘 발견했는지
  - 뭘 수정했는지
  - 왜 그렇게 판단했는지
  - 수정된 파일 목록
  - 검증 결과
