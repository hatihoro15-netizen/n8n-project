# 02. 인프라/서버 규칙

## 서버 정보
- n8n 서버: https://n8n.srv1345711.hstgr.cloud
- 로컬 개발 환경: macOS (~/n8n-project)

## 인프라 규칙
1. **IP 하드코딩 절대 금지** - 모든 IP는 설정 파일 또는 DB에서 읽기
2. **서버 직접 수정 금지** - 반드시 로컬에서 테스트 후 배포
3. **설정값 변경 시 백업 필수** - 변경 전 상태 기록
4. **슬롯 수 임의 증가 금지** - 근본 원인 파악 후 조치
5. **서버 임의 비활성화 금지** - 반드시 사유 기록 후 승인받기

## 배포 절차
1. 로컬에서 워크플로우 JSON 생성/수정
2. JSON 문법 검증 (quality-check.sh)
3. n8n 서버에 워크플로우 업로드
4. **n8n 에디터 F5 새로고침** (필수 - API 변경 반영)
5. 동작 확인
6. PROGRESS.md에 배포 기록

## 환경 변수 관리
- 하드코딩 금지
- 환경별 설정 파일 분리
- 민감 정보는 .env 파일로 관리 (.gitignore 필수)

1. 즉시 작업 중단

3. 원인 파악
4. 롤백 또는 수정
5. docs/06-error-patterns.md에 기록
