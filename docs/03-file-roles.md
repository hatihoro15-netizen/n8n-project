# 03. 파일별 역할

## 혼용 금지 원칙
- 각 파일은 하나의 역할만 담당
- 다른 역할의 코드를 섞지 않기
- 파일 역할이 불명확하면 작업 중단 후 질문
- 새 기능은 새 파일로 분리

## 파일 역할 정의

### 설정/문서 파일
| 파일 | 역할 | 수정 권한 |
|------|------|-----------|
| `CLAUDE.md` | 작업 규칙, 체크리스트 (수정 시 승인 필요) | 전체 |
| `PROGRESS.md` | 작업 기록, 현재 상태 | 전체 |
| `docs/*.md` | 매뉴얼 참고 문서 | 검증 에이전트 |

### 워크플로우 파일 (*.py)
| 파일 패턴 | 역할 | 수정 권한 |
|-----------|------|-----------|
| `create_*.py` | 워크플로우 JSON 생성 전용 | 워크플로우 에이전트 |
| `update_*.py` | 기존 워크플로우 수정 전용 | 워크플로우 에이전트 |
| `clone_and_fix.py` | 워크플로우 복제 및 수정 | 워크플로우 에이전트 |
| `gen_*.py` | 데이터/엑셀 생성 전용 | 워크플로우 에이전트 |

### JSON 파일 (*.json)
| 파일 패턴 | 역할 | 수정 권한 |
|-----------|------|-----------|
| `*_export.json` | n8n에서 내보낸 워크플로우 (읽기 전용) | 없음 |
| `*_v*.json` | 버전별 워크플로우 | 워크플로우 에이전트 |
| `*_shortform*.json` | 숏폼 영상 워크플로우 | 워크플로우 에이전트 |

### 인프라 파일
| 파일 패턴 | 역할 | 수정 권한 |
|-----------|------|-----------|
| `setup_*.sh` | 서버 셋업 스크립트 | 인프라 에이전트 |
| `docker-compose*.yml` | Docker 구성 | 인프라 에이전트 |
| `.env*` | 환경변수 (민감정보) | 인프라 에이전트 |

### 품질 관리 파일
| 파일 | 역할 | 수정 권한 |
|------|------|-----------|
| `scripts/quality-check.sh` | 품질 검사 | 검증 에이전트 |
| `.claude/commands/start.md` | 세션 시작 명령 | 검증 에이전트 |

## 수정 전 확인 사항
1. 해당 파일의 역할 확인 (이 문서 참조)
2. 자신의 에이전트 역할에 수정 권한이 있는지 확인
3. 수정 권한이 없으면 해당 에이전트에게 요청
4. 수정 전 반드시 git commit
5. export.json 파일은 절대 직접 수정 금지 (n8n에서 재내보내기)
6. 여러 파일 동시 수정 금지 → 한 파일씩 수정 후 커밋
