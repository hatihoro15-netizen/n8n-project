# 03. 파일별 역할

## 혼용 금지 규칙
- 각 파일은 하나의 역할만 담당
- 역할이 다른 로직을 같은 파일에 넣지 않기
- 새 기능은 새 파일로 분리

## 파일 유형별 역할

### Python 스크립트 (*.py)
| 파일 | 역할 |
|------|------|
| create_*.py | 워크플로우 JSON 생성 전용 |
| update_*.py | 기존 워크플로우 수정 전용 |
| clone_and_fix.py | 워크플로우 복제 및 수정 |
| gen_*.py | 데이터/엑셀 생성 전용 |
| setup.sh | 환경 설정 전용 |

### JSON 파일 (*.json)
| 파일 | 역할 |
|------|------|
| *_export.json | n8n에서 내보낸 워크플로우 (읽기 전용) |
| *_current.json | 현재 활성 워크플로우 |
| *_clean.json | 정리된 워크플로우 |
| *_v*.json | 버전별 워크플로우 |

### 문서 파일
| 파일 | 역할 |
|------|------|
| CLAUDE.md | 작업 규칙 (최우선, 수정 시 승인 필요) |
| PROGRESS.md | 작업 기록 (매 작업 후 업데이트) |
| docs/*.md | 매뉴얼 (필요 시에만 수정) |
| *.xlsx | 기획서/보고서 (읽기 전용) |

### 시스템 파일
| 파일 | 역할 |
|------|------|
| scripts/quality-check.sh | 품질 검사 스크립트 |
| .claude/commands/start.md | 세션 시작 명령 |

## 수정 시 주의사항
- export.json 파일은 절대 직접 수정하지 않음 (n8n에서 재내보내기)
- CLAUDE.md 수정 시 반드시 사용자 승인
- 여러 파일 동시 수정 금지 → 한 파일씩 수정 후 커밋
