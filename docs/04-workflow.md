# 04. 주요 작업 흐름

## 워크플로우 생성
1. 기획서(xlsx) 확인
2. 기존 유사 워크플로우 참고 (export.json)
3. Python 스크립트로 워크플로우 JSON 생성
4. quality-check.sh로 JSON 검증
5. n8n 서버에 업로드
6. 동작 확인
7. PROGRESS.md에 기록

## 워크플로우 수정
1. 현재 워크플로우 export (n8n에서 내보내기)
2. export.json 분석
3. update_*.py 스크립트로 수정
4. quality-check.sh로 JSON 검증
5. n8n 서버에 업로드
6. 기존 기능 정상 동작 확인
7. PROGRESS.md에 기록

## 워크플로우 삭제
1. 삭제 사유 기록
2. 사용자 승인 확인
3. 백업 (export.json 저장)
4. n8n 서버에서 비활성화 (즉시 삭제 금지)
5. 일정 기간 관찰 후 삭제
6. PROGRESS.md에 기록

## 의사결정 규칙

### 작업 중단 조건 (반드시 질문하고 확인 받기)
- 기존에 동작하던 기능을 수정해야 할 때
- 파일 삭제가 필요할 때
- 두 가지 방법 중 선택이 불확실할 때
- 요구사항이 모호할 때
- 예상치 못한 에러가 발생했을 때

### 판단 기준
- 확실하지 않으면 작업 중단하고 질문
- 임시방편 금지, 근본 원인 해결
- 단순한 방법이 있으면 복잡한 방법 선택 금지
- 기존 코드 스타일 유지

## 롤백 전략

### 작업 전 필수
- 반드시 git commit 후 작업 시작
- 커밋 없이 수정 시작 금지

### 문제 발생 시
1. 즉시 작업 중단
2. git diff로 변경사항 확인
3. git revert 또는 git checkout으로 롤백
4. 롤백 후 원인 파악
5. 원인 파악 완료 후에만 재시도
6. 같은 방법으로 재시도 금지 (다른 접근 방법 사용)

### 롤백 후 보고서 작성
- 무엇이 잘못됐는지
- 왜 잘못됐는지
- 다음에는 어떻게 할 것인지
- docs/06-error-patterns.md에 패턴 추가
