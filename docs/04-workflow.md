# 04-workflow.md — 작업 흐름 + 의사결정 + 롤백

## 작업 흐름
1) 요구사항 확정 → 2) 관련 docs 읽기 → 3) git commit(변경 전)
→ 4) 변경(최소 단위) → 5) quality-check → 6) 동작 확인
→ 7) git commit → 8) HANDOFF Overwrite → 9) PROGRESS 보고서 Append

## 작업 중단 조건 (반드시 질문)
- 동작 중인 기능 수정 / 파일 삭제 / 방법 선택 불확실
- 요구사항 모호 / 예상치 못한 에러 발생

## 판단 기준
- 확실하지 않으면 중단하고 질문
- 임시방편 금지, 근본 원인 해결
- 단순한 방법 우선 / 기존 코드 스타일 유지

## 롤백 전략
### 작업 전 필수
- 반드시 git commit 후 작업 시작 (커밋 없이 수정 시작 금지)

### 문제 발생 시
1. 즉시 작업 중단
2. git diff로 변경사항 확인
3. git revert 또는 git checkout으로 롤백
4. 롤백 후 원인 파악
5. 원인 파악 완료 후에만 재시도
6. 같은 방법으로 재시도 금지 (다른 접근)

### 롤백 후 필수
- 무엇이/왜/다음엔 어떻게 + docs/06에 패턴 추가
