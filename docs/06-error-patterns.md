# 06. 에러 패턴 및 해결법

## 절대 금지 패턴

### 1. IP 하드코딩
- **증상**: 코드에 직접 IP 주소 작성
- **잘못된 접근**: `host = "192.168.1.100"` 형태로 직접 입력
- **올바른 접근**: 설정 파일 또는 환경변수에서 읽기
- **재발 방지**: quality-check.sh에서 자동 탐지

### 2. 임시방편 조치
- **증상**: 슬롯 수 임의 증가, 서버 임의 비활성화
- **잘못된 접근**: 문제를 피하기 위해 설정값 임의 변경
- **올바른 접근**: 근본 원인 파악 후 정확한 수정
- **재발 방지**: 설정 변경 시 반드시 사유 기록

### 3. 근본 원인 파악 없이 재시도
- **증상**: 같은 에러가 반복 발생
- **잘못된 접근**: 단순 재실행으로 해결 시도
- **올바른 접근**: 로그 분석 → 원인 파악 → 수정 → 재시도
- **재발 방지**: 에러 발생 시 반드시 로그 확인

### 4. 파일 전체 교체
- **증상**: 수정 범위가 넓어짐
- **잘못된 접근**: 파일 전체를 새로 작성
- **올바른 접근**: 필요한 부분만 정확히 수정
- **재발 방지**: git diff로 변경 범위 확인

### 5. 동작 중인 기능 건드리기
- **증상**: 다른 기능이 깨짐
- **잘못된 접근**: 관련 없는 코드까지 수정
- **올바른 접근**: 수정 대상만 정확히 식별 후 최소 수정
- **재발 방지**: 수정 전 영향 범위 분석

### 6. 여러 파일 동시 수정
- **증상**: 롤백이 어려워짐
- **잘못된 접근**: 여러 파일을 한 번에 수정
- **올바른 접근**: 한 파일씩 수정 → 확인 → 커밋
- **재발 방지**: 파일 단위로 커밋

### 7. n8n 에디터 F5 새로고침 누락
- **증상**: API로 업로드한 변경사항이 에디터에서 저장 시 덮어써짐
- **잘못된 접근**: API 업로드 후 에디터에서 바로 작업 계속
- **올바른 접근**: API 업로드 후 반드시 에디터 F5 새로고침 → 변경 반영 확인 후 작업
- **재발 방지**: 배포 절차에 "F5 새로고침" 단계 필수 포함

### 8. n8n API PUT에 pinData 포함
- **증상**: HTTP 400 "must NOT have additional properties"
- **잘못된 접근**: GET 응답 전체를 PUT body로 전송
- **올바른 접근**: PUT body에 name, nodes, connections, settings만 포함
- **재발 방지**: update 스크립트에서 pinData 필드 제거 로직 필수

### 9. 대용량 JSON curl 인라인 전송
- **증상**: curl 명령어 실패, JSON 파싱 에러
- **잘못된 접근**: `curl -d '{"large": "json"}'` 인라인 전송
- **올바른 접근**: `curl -d @file.json` 파일 기반 전송
- **재발 방지**: update 스크립트에서 tempfile 사용

## 자주 발생하는 에러

### JSON 파싱 에러
- **증상**: `json.decoder.JSONDecodeError`
- **원인**: 잘못된 JSON 문법 (trailing comma, 따옴표 누락 등)
- **해결**: `python3 -m json.tool 파일명.json`으로 검증

### n8n 워크플로우 업로드 실패
- **증상**: 워크플로우 import 시 에러
- **원인**: 노드 ID 충돌, 필수 필드 누락, pinData 포함
- **해결**: export된 원본과 diff 비교 후 수정, pinData 제거

### Python 인코딩 에러
- **증상**: `UnicodeDecodeError`
- **원인**: 한글 파일명/내용 처리 시 인코딩 미지정
- **해결**: `open(file, encoding='utf-8')` 사용

### Google Sheets autoMapInputData 필드 불일치
- **증상**: 시트에 값이 기록되지 않음
- **원인**: 출력 필드명과 시트 컬럼명 불일치
- **해결**: 출력 JSON 키를 시트 컬럼명과 정확히 일치시키기

### FFmpeg 밈 영상 짧게 끊김
- **증상**: 밈 영상이 세그먼트 duration보다 짧게 재생
- **원인**: 원본 밈이 짧은데 루프 없음
- **해결**: 입력에 `-stream_loop -1` 추가, trim으로 duration 맞추기
