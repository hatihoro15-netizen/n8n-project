# 에러 패턴 및 해결법

## 자주 발생하는 에러

### 1. n8n API 업로드 후 에디터 덮어쓰기
- **증상**: API로 업로드한 변경사항이 에디터에서 저장 시 덮어써짐
- **잘못된 접근**: API 업로드 후 에디터에서 바로 작업 계속
- **올바른 접근**: API 업로드 후 반드시 에디터 F5 새로고침 → 변경 반영 확인 후 작업
- **재발 방지**: 배포 절차에 "F5 새로고침" 단계 필수 포함

### 2. Kie.ai 안전 필터 차단
- **증상**: 이미지/영상 생성 시 unsafe 에러
- **잘못된 접근**: 프롬프트 사전 순화 노드 추가
- **올바른 접근**: 참조 이미지 사용 시 인종/피부색 표현이 아닌 이미지 자체 문제 확인
- **재발 방지**: 에러 원인(이미지 vs 텍스트) 먼저 분리 후 대응

### 3. 상태 업데이트 row_number null 에러
- **증상**: Google Sheets 업데이트 시 row_number가 null
- **잘못된 접근**: row_number 하드코딩
- **올바른 접근**: Subject 매칭으로 변경
- **재발 방지**: 시트 업데이트는 고유 키(Subject) 기반 매칭 사용

### 4. 프레임 추출 mp4 반환
- **증상**: 프레임 추출 결과가 jpg가 아닌 mp4 URL
- **잘못된 접근**: mp4를 그대로 사용
- **올바른 접근**: `-f image2 -update 1` 옵션 추가, .mp4 URL 감지 시 원본 이미지 폴백
- **재발 방지**: 프레임 결과 URL 확장자 검증 로직 포함

### 5. n8n API GET /credentials 호출
- **증상**: 크레덴셜 목록 조회 실패
- **잘못된 접근**: GET /credentials 반복 시도
- **올바른 접근**: n8n API는 POST/DELETE만 지원, GET 미지원
- **재발 방지**: docs/02-infra-rules.md 참조

## 절대 금지 패턴

### 하드코딩
- **금지**: IP 주소, API 키, 비밀번호를 코드에 직접 입력
- **대안**: 설정 파일, 환경변수, n8n 크레덴셜 시스템 사용

### 임시방편 조치
- **금지**: 슬롯 수 임의 증가, 서버 임의 비활성화, 재시도 횟수만 늘리기
- **대안**: 근본 원인 파악 후 구조적 해결

### 근본 원인 파악 없이 재시도
- **금지**: 에러 발생 시 같은 방법으로 반복 실행
- **대안**: 에러 로그 분석 → 원인 파악 → 다른 접근 방법 시도

### 파일 전체 교체
- **금지**: 워크플로우 JSON이나 프롬프트 파일 전체를 새로 작성
- **대안**: 필요한 부분만 정확히 수정

### 동작 중인 기능 건드리기
- **금지**: 정상 동작하는 노드/연결을 "개선" 목적으로 수정
- **대안**: 문제가 있는 부분만 수정, 동작 중인 부분은 보존

### 여러 파일 동시 수정
- **금지**: 한 번에 여러 파일을 동시에 변경
- **대안**: 하나씩 수정 → 검증 → 다음 파일 수정
