# 07. 작업 보고서 양식

## 보고서 형식

```markdown
### [YYYY-MM-DD HH:MM] 작업명
- **작업 배경**: 왜 이 작업이 필요했는지
- **발견한 문제**: 작업 중 발견한 이슈
- **수정 내용**: 무엇을 어떻게 수정했는지
- **판단 근거**: 왜 이 방법을 선택했는지
- **수정된 파일**:
  - 파일경로 (변경 내용 요약)
- **검증 결과**: quality-check.sh 통과 여부, 동작 확인 결과
- **다음 권장 작업**: 후속으로 해야 할 작업
```

## 작성 규칙
1. 모든 작업 완료 후 PROGRESS.md에 위 형식으로 기록
2. 작업명은 구체적으로 (예: "워크플로우 노드 분리" O, "작업" X)
3. 수정된 파일은 전체 경로로 기록
4. 검증 결과는 PASS/FAIL + 상세 내용
5. 다음 권장 작업은 구체적 행동으로 기술

## 에이전트 보고서 추가 항목
에이전트 작업 시 아래 항목 추가:
```markdown
- **에이전트 역할**: (인프라/워크플로우/콘텐츠/검증)
- **담당 범위 확인**: 자기 역할 내 파일만 수정했는지 (예/아니오)
```

## PROGRESS.md 기록 위치
- `## 작업 기록` 섹션 하단에 최신 기록을 추가
- 동일 주제 반복 기록은 최신 1개만 유지 (이전 기록 삭제)
