# 작업 보고서 양식

## 템플릿

```
[YYYY-MM-DD HH:MM] 작업명
* 작업 배경:
* 발견한 문제:
* 수정 내용:
* 판단 근거:
* 수정된 파일:
* 검증 결과:
* 다음 권장 작업:
```

## 작성 규칙
- 모든 작업 완료 후 PROGRESS.md에 위 형식으로 기록
- 간결하게 핵심만 작성 (장황한 설명 금지)
- 수정된 파일은 정확한 경로로 기재
- 검증 결과는 quality-check.sh 통과 여부 포함
- 다음 권장 작업이 없으면 "없음" 기재

## 예시

```
[2026-03-01 04:00] 프로젝트 구조 정리
* 작업 배경: 불필요한 구버전 파일이 42개 존재, 프로젝트 구조 복잡
* 발견한 문제: v4~v6 온카스터디 파일, 구버전 워크플로우 JSON 17개, 일회성 Python 스크립트 16개 잔존
* 수정 내용: 불필요 파일 42개 삭제, 캐릭터 이미지 characters/ 폴더로 이동, 기획서 docs/ 폴더로 이동
* 판단 근거: v8.4가 최신 버전이므로 이전 버전 파일 불필요
* 수정된 파일: 42개 삭제, characters/ 생성, docs/ 정리
* 검증 결과: ls -la로 최종 구조 확인 완료
* 다음 권장 작업: 매뉴얼 시스템 구축
```
