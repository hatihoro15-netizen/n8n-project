## 실패체크 / 폴백 로직

### 영상 생성 2단계 방어

```
1단계: 영상 생성 폴링 (실패체크)
    ├─ 성공 (successFlag === 1) → 다음 단계
    ├─ 진행중 → 30초 재대기 → 재폴링 (최대 20회 = 10분)
    └─ 실패 확정 (flag >= 2 또는 20회 초과) → 원본 이미지 폴백

2단계: 폴백 처리
    ├─ successFlag를 1로 위장 → 완료체크 통과
    ├─ 프레임 추출 스킵 (videoFailed 감지)
    ├─ 다음 영상은 원본 이미지 폴백 사용
    └─ NCA에서 해당 영상 스킵 (URL 없음)
```

- 영상N 생성: `onError: continueRegularOutput` 설정 (400 에러 시 워크플로우 중단 방지)
- 영상N 결과: `onError: continueRegularOutput` 설정

### staticData 사용
- `retryCount` (실패체크): 폴링 횟수 (20회 초과 시 타임아웃 판정, 성공/실패 시 리셋)

- `imageComplete` 플래그 출력 (true/false)
- staticData로 retryCount 관리 (30회 초과 시 타임아웃 에러)
- **하드코딩 API 키 제거** → 이미지 결과 HTTP Request 노드의 크레덴셜로 폴링

### 영상N 준비
- 영상1: 이미지 URL 추출 결과 사용
- 영상2: 프레임1 결과의 `frameUrl` 사용 (없으면 원본 이미지 폴백)
- 영상3: 프레임2 결과의 `frameUrl` 사용 (없으면 원본 이미지 폴백)
- 출력: `veo3Body` (prompt, imageUrls, model=veo3_fast, 16:9)

### 프레임N 추출
- NCA FFmpeg API로 7.5초 시점 프레임 추출
- `videoFailed` 시 스킵 → `skipped: true` 반환
- **videoUrl 패스스루** (재진입 시 재사용)
- 재진입: `data.videoUrl` 우선, 없으면 `data.response.resultUrls[0]`

### 프레임N 결과
- NCA 응답에서 `response[0].file_url` 추출
- `frameComplete` 플래그 출력 (true/false)
- staticData로 retryCount 관리 (10회 초과 시 원본 이미지 폴백)
- 재진입 시 NCA 재요청 (videoUrl 사용)

### NCA 데이터 준비
- 영상1~3 결과에서 URL 수집 (없으면 스킵)
- 클립 길이: 8초 (`CLIP_DURATION = 8`)
- FFmpeg: scale 1920x1080, pad black, 30fps, concat
- 출력: H.264 CRF 18, AAC 256k, yuv420p, faststart
- SRT 자막 생성 (장면별 대사 + 타이밍)

### NCA 결과 처리
- 최종 영상 URL 추출 (array/string/object 대응)
- subject, caption, comment, SRT 데이터 패스스루
